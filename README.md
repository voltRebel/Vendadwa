# Vendora

Vendora is a multi-tenant PHP point-of-sale (POS) and inventory management system designed to run on a typical XAMPP/LAMP stack. This repository contains the web application and database migration scripts for setting up and managing companies, branches, users, products, sales, expenses, backups and more.

---

## Requirements

- **PHP**: 7.4+ (recommended 8.0+)
- **Web server**: Apache or Nginx (XAMPP on Windows is fine)
- **Database**: MySQL or MariaDB
- **Extensions**: `pdo_mysql`, `mbstring`, `json`, `openssl`

This guide assumes you are running on **Windows with XAMPP**, but the same steps apply to any LAMP stack with minor path changes.

---

## Local Installation (XAMPP)

- **1. Clone or copy the project into your web root**
  - For XAMPP on Windows, place the project inside `C:\xampp\htdocs\`
  - Example path used here: `C:\xampp\htdocs\Roach's Workshop\Vendora`

- **2. Create the database**
  - Start **Apache** and **MySQL** from XAMPP Control Panel.
  - Open `http://localhost/phpmyadmin`.
  - Create a new database (e.g. `vendora_db`) with collation `utf8mb4_unicode_ci`.

- **3. Configure database connection**
  - Open `includes/db.php`.
  - Update the following values to match your environment:

```php
$db_host = 'localhost';
$db_name = 'vendora_db';   // your database name
$db_user = 'root';         // your DB user
$db_pass = '';             // your DB password (empty by default on XAMPP)
```

- **4. Run the database setup / migration**
  - In your browser, navigate to:
    - `http://localhost/Roach's%20Workshop/Vendora/setup_database.php`
  - This script will:
    - Create or update all required tables (companies, users, products, sales, etc.).
    - Create a default **Super Admin** user.
    - Create a test **Vendora** company and an **Admin** user for it (if needed).

- **5. Default login credentials**
  - **Super Admin**
    - Username: `Roach`
    - Password: `flying@DUTCHMAN97`
  - **Company Admin (Vendora)**
    - Username: `admin`
    - Password: `password123`

  > You should change these passwords immediately after first login.

- **6. Accessing the application**
  - Main entry point:
    - `http://localhost/Roach's%20Workshop/Vendora/index.php`
  - Login page (if routed by `index.php`):
    - `http://localhost/Roach's%20Workshop/Vendora/index.php?page=login`

## Project Structure (High Level)

- `index.php` — Main front controller / router.
- `setup_database.php` — One-off (or repeatable) script to create/migrate database for multi-tenancy.
- `includes/`
  - `db.php` — Database connection (PDO).
  - `header.php`, `footer.php`, `sidebar.php` — Layout/partials.
  - `queries.php` — Common database queries/helpers.
- `controllers/` — Application logic (e.g. `POSController.php`, `ProductController.php`, `AuthController.php`, etc.).
- `pages/` — UI pages, including:
  - `dashboard.php`, `pos.php`, `products.php`, `customers.php`, `suppliers.php`, `expenses.php`, `returns.php`, `reports.php`, `settings.php`, `tools.php`.
  - `auth/` — Authentication pages (`login.php`, `forgot_password.php`, etc.).
  - `super_admin/` — Super admin pages (e.g. companies management, dashboard).
- `assets/`
  - `css/style.css` — Main styles.
  - `js/app.js` — Front-end scripts.
- `backups/` — SQL backup files generated by the app (ignored by Git).
- `database/migrations/` — SQL migration helpers (e.g. `create_backups_table.sql`).

## Running Database Migrations Later

If you make schema-related changes or pull new updates that affect the database:

- Re-run:
  - `http://localhost/Roach's%20Workshop/Vendora/setup_database.php`
- Or, if provided, use:
  - `http://localhost/Roach's%20Workshop/Vendora/run_migration.php`

The scripts are designed to be idempotent, adding missing columns/tables safely when needed.

## Git & GitHub Setup

### 1. Initialize Git in this folder (already done once)

From the project root (`Vendora` folder), run:

```bash
git init
git add .
git commit -m "Initial Vendora POS setup"
```

> If you are reading this in the GitHub repo, the above commands were already run for the first commit.

### 2. Connect to your GitHub repository

In a terminal inside the project folder:

```bash
git branch -M main
git remote add origin https://github.com/<YOUR_GITHUB_USERNAME>/Vendora.git
git push -u origin main
```

Replace `<YOUR_GITHUB_USERNAME>` with your actual GitHub username.

If you prefer SSH:

```bash
git remote add origin git@github.com:<YOUR_GITHUB_USERNAME>/Vendora.git
git push -u origin main
```

## Common Customizations

- **Change default company info**
  - Update records in the `companies` table (via phpMyAdmin) or through the Settings page in the app.

- **Change login credentials**
  - After first login, go to the Users/Settings section and update the Super Admin/Admin passwords.

- **Branding**
  - Replace logos and tweak colors in:
    - `assets/css/style.css`
    - Any logo paths stored in the `companies` table.

---

## Security Notes

- Do **not** deploy with the default passwords in production.
- Protect `setup_database.php` and `run_migration.php`:
  - Delete them, move them outside the web root, or restrict access once setup is complete.
- Ensure proper file permissions on the server and avoid exposing `backups/` publicly.
